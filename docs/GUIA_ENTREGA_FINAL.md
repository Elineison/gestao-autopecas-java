# Guia de Entrega Final - Uniasselvi

Este guia organiza o que precisa ser feito para transformar o projeto Gestão AutoPeças em uma entrega final completa para a disciplina Imersão Profissional: Implementação de uma Aplicação.

## Situação Atual

O projeto já possui uma aplicação funcional em Java com Spring Boot, interface web, cadastro de produtos, cadastro de clientes, registro de vendas, baixa automática no estoque e listagem de vendas recentes.

O paper também já está estruturado com os tópicos exigidos no modelo da Uniasselvi:

- Resumo;
- Introdução;
- Fundamentação Teórica;
- Metodologia;
- Considerações;
- Referências Bibliográficas.

## O Que Falta Para Entregar

Antes de enviar o trabalho, faça estes ajustes finais:

- preencher nome do acadêmico, turma e tutor no paper;
- passar o conteúdo do arquivo `docs/PAPER.md` para o modelo oficial `modelo-paper-uniasselvi.docx`;
- inserir os prints reais da aplicação nos pontos indicados no paper;
- conferir se o resumo ficou com até 250 palavras;
- revisar fonte, espaçamento e alinhamento conforme o modelo;
- montar a apresentação usando `template-socializacao-uniasselvi.pptx`;
- usar o conteúdo de `docs/PPT_SOCIALIZACAO.md` como base dos slides;
- testar a aplicação antes da apresentação.
- conferir se o link do GitHub está correto no paper e no PPT.

## Prints Que Devem Entrar No Paper

Use prints simples, com a tela limpa e legível. Não precisa colocar muitos prints repetidos. O ideal é mostrar o fluxo principal.

1. Tela inicial da aplicação.
2. Cadastro de produto preenchido.
3. Cadastro de cliente preenchido.
4. Registro de venda preenchido.
5. Lista de estoque após a venda.
6. Lista de vendas recentes.

Sugestão: no paper, use de 4 a 6 imagens. No PPT, use menos imagens e deixe a tela mais visual.

## Ordem Recomendada No Paper

Use esta ordem no documento final:

1. Título, acadêmico, turma e tutor.
2. Resumo e palavras-chave.
3. Introdução.
4. Fundamentação teórica.
5. Trecho de código da baixa de estoque.
6. Metodologia.
7. Trecho de código do registro de venda.
8. Prints da aplicação.
9. Considerações.
10. Referências bibliográficas.

## Pontos Fortes Do Trabalho

Estes são os pontos que devem aparecer com clareza na escrita e na apresentação:

- o tema resolve uma necessidade comum de pequenos comércios;
- a aplicação foi feita em Java, alinhada com a disciplina;
- o sistema tem regra de negócio real, não apenas telas;
- a venda altera automaticamente o estoque;
- a aplicação pode ser testada localmente;
- o código foi separado em camadas, facilitando entendimento e manutenção.
- o código-fonte está organizado no GitHub: `https://github.com/Elineison/gestao-autopecas-java`.

## Cuidados Para Não Parecer Texto Artificial

Evite frases muito genéricas, como:

- "solução inovadora";
- "ferramenta robusta";
- "sistema altamente escalável";
- "otimização de processos empresariais";
- "ecossistema tecnológico".

Prefira frases diretas, como:

- "A aplicação ajuda a controlar produtos, clientes e vendas";
- "A baixa automática evita que o estoque seja atualizado manualmente";
- "O projeto foi limitado às funções principais para atender à proposta da disciplina";
- "O sistema pode ser executado localmente e testado pela interface web".
